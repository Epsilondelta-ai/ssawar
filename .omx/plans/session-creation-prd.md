# ssawar 세션 생성 PRD

작성일: 2025-02-14
상태: Draft v1

## 목적

`ssawar`의 첫 진입 화면과 `채팅 세션 생성` 흐름을 제품 요구사항 수준으로 고정한다.

이 문서의 목적은 아래 4가지를 결정하는 것이다.

1. 유저가 세션 시작 전에 실제로 무엇을 입력하는가
2. 무엇을 기본값으로 자동 채우고, 무엇을 고급 설정으로 숨기는가
3. 세션 생성 직후 어떤 상태로 라이브 세션에 진입하는가
4. 세션 제목이 어떻게 정해지고 수정되는가

## 배경

현재 상위 기획에서 이미 정해진 사실은 다음과 같다.

- `ssawar`는 대외적으로는 `AI들끼리 싸움을 붙이는 서비스`로 보인다.
- 내부적으로는 범용 오케스트레이터 엔진 위에서 돌아간다.
- `오케스트레이터 모델`은 유저가 `채팅 세션 생성 시점`에 선택한다.
- 참가 AI는 최대 16개까지 지원한다.
- 세션 생성 시 질문은 필수가 아니다.
- 세션은 기본 제목 `Untitled`로 생성되고, 대화 내용으로 제목이 자동 생성된다.

문제는, 이 요구를 그대로 노출하면 세션 생성 화면이 무거워질 수 있다는 점이다.
따라서 세션 생성 UX는 `질문 없이도 시작 가능`, `선택은 필요한 것만`, `나머지는 추천값` 원칙으로 설계한다.

## 목표

### 제품 목표
- 유저가 1분 이내에 첫 세션을 열 수 있어야 한다.
- 세션 생성은 가볍게 느껴져야 하지만, 핵심 제어권은 유지해야 한다.
- 오케스트레이터 모델 선택은 유지하되, 다른 설정은 대부분 숨긴다.
- 세션 생성 직후에는 ChatGPT처럼 빈 채팅 화면으로 진입해야 한다.

### 사용자 목표
- 질문이 없어도 세션을 먼저 만들고 바로 대화를 시작하고 싶다.
- 원하면 오케스트레이터와 참가 AI를 직접 바꿔볼 수 있다.
- 복잡한 규칙을 공부하지 않아도 바로 시작할 수 있어야 한다.

## 비목표

- 세션 생성 단계에서 모든 모드 설정을 노출하지 않는다.
- 유저 정의 규칙 생성기는 다루지 않는다.
- 마피아 같은 private-state 모드 전용 설정은 다루지 않는다.
- 결제, 크레딧, 팀 협업 설정은 이 단계에서 다루지 않는다.

## 핵심 원칙

1. 질문 없이도 세션을 만들 수 있어야 한다.
2. 오케스트레이터 모델은 선택 가능해야 하지만, 선택 UI는 짧아야 한다.
3. 참가 AI는 추천 조합을 기본으로 채우고, 직접 수정은 가능하게 한다.
4. 고급 설정은 기본적으로 닫혀 있어야 한다.
5. `시작`까지 필요한 인지 부하는 최소화한다.
6. 세션 제목은 생성 시 강제하지 않고, 대화 후 자동 생성한다.

## 핵심 사용자 유형

### 1. 빠른 구경형
- 질문 없이도 빈 세션을 먼저 만들고 바로 말을 걸고 싶다.
- 상세 설정을 거의 보지 않는다.

### 2. 모델 비교형
- 오케스트레이터 모델과 참가 AI를 일부 바꿔보고 싶다.
- 기본 추천 조합을 바탕으로 1~2개만 손본다.

### 3. 실험형
- 참가 AI를 많이 넣고, 세션 성격을 바꿔 보길 원한다.
- 고급 설정을 사용한다.

## 사용자 시나리오

### 시나리오 A. 초간단 시작
1. 유저가 홈에서 `새 세션`을 누른다
2. 시스템은 기본 오케스트레이터 모델을 미리 선택해 둠
3. 시스템은 추천 참가 AI 2~4개를 자동 채움
4. 유저가 `시작`을 눌러 즉시 빈 세션 진입
5. 세션 제목은 `Untitled`

### 시나리오 B. 오케스트레이터만 바꾸기
1. 유저가 세션 생성 패널을 연다
2. 오케스트레이터 모델 드롭다운 또는 모델 피커를 열어 다른 모델 선택
3. 참가 AI 추천 조합은 그대로 둠
4. `시작`을 눌러 세션 진입

### 시나리오 C. 참가 AI 직접 편집
1. 유저가 세션 생성 패널을 연다
2. 오케스트레이터 모델 선택
3. 추천 참가 AI 중 일부를 제거하고 새로운 AI를 추가
4. 고급 설정은 열지 않고 바로 시작

## 화면 범위

이 PRD는 아래 화면만 다룬다.

- 홈의 `새 세션 + 빠른 시작` 영역
- `채팅 세션 생성` 패널 또는 페이지
- 세션 생성 직후 라이브 세션 진입 직전 로딩 상태

## 세션 생성 UX 구조

### 옵션 1. 인라인 홈 생성
- 홈 첫 화면에서 `새 세션` 클릭
- 클릭 후 하단 패널이 확장되며 오케스트레이터와 참가 AI 설정이 노출
- 가장 빠른 흐름

### 옵션 2. 모달/시트 기반 생성
- 홈에서 `새 세션`을 누르면 생성 시트 오픈
- 오케스트레이터, 참가 AI, 고급 설정을 여기서 조정
- 모바일 대응이 쉽다

### 권장안
- 데스크탑: 인라인 + 우측 설정 패널
- 모바일: 바텀시트 기반 생성

## 입력 항목 정의

### 1. 첫 메시지
- 선택
- 세션 생성 단계에서는 비워둘 수 있다
- 실제 첫 메시지는 세션 진입 후 채팅 입력창에서 보낸다
- 세션 생성 단계의 필수값이 아니다

### 2. 오케스트레이터 모델 선택
- 필수
- 세션 생성 시 항상 보인다
- 단, 기본 추천 모델이 미리 선택돼 있어야 한다
- 유저는 필요 시 바꿀 수 있다

### 3. 참가 AI 선택
- 필수
- 최소 참가 수는 배틀 모드 기준 2
- 최대 16
- 기본 추천 조합 2~4개 자동 채움
- 유저는 제거/추가 가능

### 4. 진행 방식
- 선택
- 기본값은 `짧게`
- 고급 설정에서 노출

### 5. 최대 턴 수
- 선택
- 기본값은 6
- 고급 설정에서 노출

### 6. 공개 범위
- 선택
- 기본값은 `비공개`
- 고급 설정에서 노출

## 기본값 정책

### 세션 생성 직전
- 오케스트레이터 모델: 시스템 추천 기본값 미리 선택
- 참가 AI: 기본 조합 미리 보임
- 세션 제목: `Untitled`

### 첫 메시지 입력 후
- 시스템은 대화의 성격을 가볍게 분석해 세션 제목 후보를 생성할 수 있다
- 시스템은 메시지의 성격을 바탕으로 추천 참가 AI 로직을 고도화할 수 있다
- 예:
  - 창업/비즈니스 질문 -> 논리형 모델 조합
  - 밈/드립 질문 -> 유쾌한 모델 조합

### 시작 버튼 직전
- 유저가 별도 조정하지 않으면 기본 추천값으로 그대로 시작

## 참가 AI 선택 UX

### 기본 UX
- 추천 참가 AI 칩 2~4개를 먼저 보여준다
- `+ AI 추가`를 눌러 검색형 선택 UI를 연다
- 선택된 AI는 토큰형 태그로 나열한다

### 16개 지원 UX
- 5개 이상부터는 가로 태그 나열 대신 접힘/펼침 처리
- `+12 more` 같은 축약 표기를 둔다
- 라이브 세션 진입 시에는 전체 카드가 아니라 요약 배치 또는 그룹 뷰를 사용한다

## 오케스트레이터 모델 선택 UX

### 요구사항
- 유저가 세션 생성 시 직접 선택 가능해야 한다
- 하지만 선택 부담은 작아야 한다

### 권장 UX
- 기본값이 미리 선택된 segmented control 또는 compact dropdown
- `추천` 배지 표시
- `이 오케스트레이터가 하는 일` 툴팁 제공

### 설명 문구 예시
- `진행자 AI`
- `이 AI가 세션 흐름, 질문, 정리 타이밍을 관리한다`

## 시작 버튼 동작

### 버튼 라벨
- `바로 시작`
- 또는 `붙여보기`

### 클릭 시
1. 필수값 검증
2. 세션 생성 API 호출
3. 오케스트레이터 초기 세팅 결정
4. `Untitled` 제목의 빈 라이브 세션으로 이동

### 로딩 상태
- `AI들 모으는 중...`
- `오케스트레이터가 판을 짜는 중...`

## 검증 규칙

### 첫 메시지
- 세션 생성 시에는 비어 있어도 시작 가능
- 첫 메시지는 세션 진입 후 자유롭게 입력

### 오케스트레이터 모델
- 필수
- 선택 안 되어 있으면 시작 불가

### 참가 AI
- 최소 2개 필요
- 최대 16개 초과 불가
- 중복 선택 불가

## 예외 처리

### 오케스트레이터 모델 로드 실패
- 기본 fallback 모델 자동 선택
- 유저에게 짧은 토스트 표시

### 참가 AI 일부 로드 실패
- 추천 조합에서 실패 모델 제거
- 시작은 계속 가능

### 생성 API 실패
- 입력값은 유지
- 재시도 버튼 제공

## 라이브 세션 진입 직후 요구사항

- 첫 진입 시 현재 선택된 오케스트레이터 모델이 보여야 한다
- 참가 AI 전체가 로드되기 전까지 skeleton 또는 joining 상태를 보여야 한다
- 오케스트레이터의 첫 안내 문장이 1회 보여야 한다
- 세션 제목은 기본적으로 `Untitled`여야 한다
- 대화 내용으로 제목이 자동 생성돼야 한다
- 유저는 제목을 언제든 수정할 수 있어야 한다

## 모바일 요구사항

- 새 세션 CTA는 상단에서 바로 접근 가능해야 한다
- 오케스트레이터 선택은 한 줄 드롭다운 또는 시트 버튼
- 참가 AI 선택은 검색 시트로 분리
- 고급 설정은 기본 접힘

- 새 세션 버튼 클릭률

- 자동 제목 생성 노출률

## Acceptance Criteria

1. 유저는 질문 없이도 1분 내 첫 세션을 시작할 수 있어야 한다.
2. 오케스트레이터 모델 선택 UI는 세션 생성 화면에 항상 보여야 한다.
3. 오케스트레이터 모델은 기본 추천값이 미리 선택돼 있어야 한다.
4. 참가 AI는 추천 조합이 자동으로 채워져 있어야 한다.
5. 참가 AI는 최소 2개, 최대 16개까지 선택 가능해야 한다.
6. 유저는 고급 설정을 열지 않고도 세션을 시작할 수 있어야 한다.
7. 세션 생성 직후 제목은 `Untitled`여야 한다.
8. 대화 내용으로 제목이 자동 생성돼야 한다.
9. 유저는 제목을 수동 수정할 수 있어야 한다.
10. 모바일에서도 오케스트레이터 선택, 참가 AI 편집, 시작이 가능해야 한다.
11. 세션 생성 실패 시 입력값이 유지돼야 한다.

## Out of Scope

- 참가 AI 자동 밸런싱 알고리즘 고도화
- 모드별 전용 세션 생성 화면 분기
- private-state 모드용 생성 폼
- 크레딧/요금제 선택 UX
